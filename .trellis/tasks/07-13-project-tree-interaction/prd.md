# 项目/树/新建/关闭交互

## Goal

优化项目级交互全流程:欢迎页最近项目显示中文名、新建项目对话框、分组树拖拽与 select-none、新建分组/表正经对话框、节点显示、表复制、关闭项目回欢迎页、未保存提醒。

## Requirements

1. **欢迎页最近项目**:路径之上显示项目中文名(无中文名 fallback basePackage 或文件名)
2. **新建项目对话框**:弹对话框输入项目名(中文)+ basePackage(英文包名),确认后创建(替代直接创建空项目)
3. **左侧树 select-none + 拖拽**:
   - 分组排序(分组间拖拽)
   - 表跨分组移动(表拖到另一分组)
   - 表组内排序(表在分组内拖拽)
   - 层级约束:分组只能顶层,表只能在分组下
4. **新建分组/表正经对话框**(替代 prompt):
   - 分组:code + 中文名
   - 表:code + 中文名
5. **节点显示**:分组只显示中文名(去 code);表显示 `code (中文名)`
6. **表复制**:复制表结构为新表(code 加 `_COPY` 后缀,重复则数字递增;name 加"(副本)";字段/索引深拷贝;加入同分组)
7. **关闭项目回欢迎页**:菜单"文件"加"关闭项目"项,关闭后回欢迎页(currentProject=null)
8. **未保存提醒**:改动未保存时,关闭项目/打开新项目提醒"是否保存"(保存/不保存/取消)

## Acceptance Criteria

- [ ] 欢迎页最近项目列表每项显示中文名(路径之上)
- [ ] 新建项目弹对话框填项目名+basePackage
- [ ] 树 select-none;可拖拽分组排序、表跨分组、表组内排序;层级约束生效
- [ ] 新建分组/表用对话框(分 code+中文名字段),不再用 prompt + `|` 分隔
- [ ] 分组节点只显中文名;表节点显 `code (中文名)`
- [ ] 表复制生成新表(code 后缀、name 副本、字段索引拷贝)
- [ ] 关闭项目回欢迎页
- [ ] 改动未保存时,关闭项目/打开新项目弹"是否保存"(三选项)
- [ ] pnpm build 通过

## Notes

- 未保存 dirty 用简单 watch(deep)实现;Undo/Redo 任务后续重构 dirty 基于历史栈。
- 表复制不复制数据(只结构)。
- 退出应用(关窗口)的未保存提醒:Tauri close 事件,本任务可先不做(若复杂),聚焦关闭项目/打开新项目。
